About

As part of my computer science A-Level I am required to create a project to hand in as coursework. I chose to base mine around path finding algorithms. Over the next few months I will be creating a GUI within Python (using the Pygame module) to emulate how both Dijkstra's and A* algorithms find the shortest path between two nodes within a grid The program will allow the user to create their own environment and implement whichever algorithm they wish.

The project

Analysis

The analysis is the first section of my coursework, it is where I layed my initial ideas for the project, describing the intended function and the end user that will use it. Within the analysis I also had to go indepth into how both Dijkstra's and A* algorithms work, explaining why it will be a complex problem to solve.

Design

The next section to complete is the Design, in this I go more indepth into how I will create my program, writing sample pseudocode to show how I would implement my core algorithms, as well as individual features such as Object Oriented Programming and using Pygame. Furthermore, it requires me to break down the system into individual sections to show an overview of how it will work

Development

In development I will be keeping a log of my creation of the program, seperating it into versions to show how I have developed it over time. Within this I have decided to split my deveopment into two areas: the creation of the GUI and the creation of the path finding algorithms. This will allow me to view and test the two areas individually before implementing them within each other

Testing

Testing will consist of the use of trace tables to ensure all functions work as intended, and if not to highlight the issue it faces.

Evaluation

The evaluation is where I compare my overall system against the aims and objectives I had set within the Analysis. A successful and well implemented program should meet as many (if not all) of these objectives as possible.

Dijkstra's algorithm

Dijkstra's algorithm is a very basic path finding algorithm. It will check every route possible before agreeing on the shortest path possible. Learn more.

A* algorithm

A* is a path finding algorithm that can be more quick and efficient. It uses heuristics to find a suitable route to the destination (this isn't necessarily the quickest path), predicting the distance between each node visited and the finish goal. Learn more.
